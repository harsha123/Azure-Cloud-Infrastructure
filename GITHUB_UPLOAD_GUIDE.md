# GitHub Upload Guide

This package is prepared to update your GitHub profile according to your Azure Cloud Infrastructure Engineer CV.

## Option 1 — Update your GitHub profile README

1. Log in to GitHub.
2. Create a new repository with the **same name as your GitHub username**.
   - Example: if your GitHub username is `harsha-tennakoon`, create a repository named `harsha-tennakoon`.
3. Keep it **Public**.
4. Upload the root `README.md` file from this package.
5. Commit with message: `Update GitHub profile README`.

## Option 2 — Upload the project folders

Create these public repositories, or upload them as folders into one portfolio repository:

1. `azure-infrastructure-operations-blueprint`
2. `hybrid-connectivity-network-support`
3. `azure-monitoring-backup-resilience`
4. `powershell-azure-ops-toolkit`

For each folder:

1. Create a new GitHub repository with the same folder name.
2. Upload the files inside that folder.
3. Commit with a clear message, for example: `Add Azure infrastructure operations blueprint`.

## Important public GitHub safety note

Do not upload:

- Personal phone number
- Home address
- Passport, visa or immigration documents
- Customer names, internal IPs, production screenshots or secrets
- Real subscription IDs, tenant IDs, client IDs, passwords, certificates or access keys

Use placeholders such as:

- `<subscription-id>`
- `<tenant-id>`
- `<resource-group-name>`
- `<company-name>`
